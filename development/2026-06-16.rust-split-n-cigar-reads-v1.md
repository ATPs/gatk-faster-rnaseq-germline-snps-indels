# Rust SplitNCigarReads v1 fast mode

## Aim

Develop roadmap module 3: a first Rust replacement for GATK `SplitNCigarReads` that can be selected from the existing Python step wrapper while keeping the default backend on GATK.

## Scope

This implementation started as v1 fast mode. It now also has an opt-in compatibility mode for closer tag behavior and reference-based overhang fixing.

Implemented:

- split primary RNA-seq reads at `N` CIGAR operations;
- preserve soft clips, hard clips, insertions, deletions, mate fields, flags, read groups, sequence, and qualities;
- apply the default STAR mapping-quality transform `MQ 255 -> 60`;
- skip secondary alignments by default, with an explicit option to process them;
- remove stale `NM`, `MD`, `NH`, `MC`, and `SA` tags from fast-mode split records;
- mark additional split records as supplementary;
- maintain coordinate-sorted output with a priority queue and write a BAI index;
- expose the Rust backend through `step_split_n_cigar.py --backend rust`;
- expose the pipeline backend through `run_pipeline.py --split-n-cigar-backend rust`;
- expose Rust modes through the CLI `--mode fast|compatibility`, the step wrapper `--rust-mode fast|compatibility`, and the pipeline `--split-n-cigar-rust-mode fast|compatibility`.

Additional opt-in compatibility mode behavior:

- keep `MC` and `SA` tags instead of stripping them like fast mode;
- repair `MC` by splitting the mate CIGAR and keeping the first split segment, matching GATK's direct mate-CIGAR repair path when no overhang prediction is available;
- remove stale `NM`, `MD`, and `NH` tags from split and unsplit records, matching GATK's cleanup list;
- reconstruct split-family `SA` tags using header contig names when available;
- preserve existing `SA` entries and append split-family entries in GATK-like order;
- repair `MC` even when secondary alignments are skipped by default;
- run a two-pass compatibility path with splice tracking, reference mismatch checks, overhang soft clipping, and mate-position/`MC` prediction repair.

Still deferred beyond the selected-window v2 compatibility checks:

- full-sample BAM-level equivalence testing;
- full equivalence for RNA-seq read filters beyond the current `AllowAllReads` default behavior.

## Changes

- Integrated `rust_split_n_cigar_reads` into the existing Rust crate under `src/rust/`.
- Added split logic in `src/rust/src/split_n_cigar.rs` and a CLI in `src/rust/src/bin/rust_split_n_cigar_reads.rs`.
- Added CIGAR and record-level unit tests for `N`, `S`, `H`, insertions, deletions, paired reads, supplementary reads, secondary reads, unmapped-mate flags, compatibility-mode `SA` reconstruction, compatibility-mode `MC` repair, and reference-driven left/right overhang clipping.
- Added Python wrapper/backend arguments without changing the default GATK behavior.
- Added `src/rust/.cargo/config.toml` so `hts-sys` bindgen can find conda `libclang` and system C headers in this environment.

## Conclusion

Module 3 v1 fast mode is implemented and callable from the step wrapper and pipeline. On SRR949115 it is `51.6x` faster than the measured GATK `SplitNCigarReads` step and passed the downstream fast-mode VCF acceptance thresholds when followed by GATK BQSR and scattered GATK `HaplotypeCaller`.

This is ready for opt-in speed-first adoption through `--split-n-cigar-backend rust`. Compatibility mode is available for stricter tag and overhang behavior through `--split-n-cigar-rust-mode compatibility`, but the default pipeline backend should remain GATK until compatibility mode has been validated on a full sample.
