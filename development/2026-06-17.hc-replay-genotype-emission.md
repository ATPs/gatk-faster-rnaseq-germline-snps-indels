# HaplotypeCaller replay genotype emission parity fix

## Aim

Make replay genotype output reflect the same final assembled-event calls as the Rust production HaplotypeCaller path.

## Source check

Local source verification used:

- Production assembled-event genotyping in `src/rust/src/haplotype_caller.rs`
- Replay region scan/output flow in `src/rust/src/haplotype_caller.rs`

Confirmed findings:

- Replay already assembled haplotypes and wrote candidate event rows plus PairHMM rows.
- Replay genotype tables were built from `output.variants`.
- `scan_replay_window` extended `output.variants` with `region_events`, but `region_events` was never populated.
- Production had the actual assembled-event genotyping logic inline and pushed its final calls directly into `output.variants`.

Conclusion from source: replay diagnostics were not showing the same final calls that production would emit for the same region.

## Code changes

Changed file:

- `src/rust/src/haplotype_caller.rs`

Implementation:

- Extracted the assembled-event genotyping logic into a shared helper:
  - `genotype_assembled_events(...)`
- Switched production to use that helper when converting PairHMM haplotype likelihoods into final variant calls.
- Switched replay to:
  - collect the same read-by-haplotype likelihood matrix it already computes for PairHMM rows;
  - call the same shared genotyping helper;
  - append the returned final calls to `output.variants`.
- Removed the unused replay-only `region_events` placeholder path.
- Added a focused unit test that verifies the shared helper emits a final call from a simple synthetic PairHMM matrix.

## Conclusion

This removes a replay/production divergence that made genotype-diff diagnostics less trustworthy.

The next useful validation is to run replay/genotype diff tooling on selected regions or rerun the next full Java-vs-Rust good-set comparison. The next remaining code candidate in the aim file is active-region padding, but that should be driven by sampled boundary-miss evidence rather than changed blindly.
