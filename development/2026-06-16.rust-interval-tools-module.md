# rust_interval_tools module 1

## Aim

Implement module 1 from the optimization roadmap: a Rust interval preparation tool that replaces the expensive GATK `BedToIntervalList` and `SplitIntervals` path for exon interval preparation.

## Changes

- Added `rust_interval_tools` to the single Rust crate under `src/rust/`, with shared code in `src/rust/src/` and the binary entrypoint in `src/rust/src/bin/`.
- Added the `rust_interval_tools` binary with three subcommands:
  - `bed-to-interval-list`
  - `split-intervals`
  - `prepare`
- The tool parses BED and Picard/GATK `interval_list` files, validates intervals against a sequence dictionary, sorts by dictionary order, merges overlapping/bookended intervals by default, preserves GATK-compatible interval_list headers, and writes balanced `*-scattered.interval_list` shards.
- Added `--backend gatk|rust` and `--rust-bin` to:
  - `src/step_bed_to_interval_list.py`
  - `src/step_split_hc_intervals.py`
- Added `--interval-backend gatk|rust` and `--rust-interval-tools` to `src/run_pipeline.py`. The default remains GATK.

- Invalid required input check exited non-zero as expected.

Real-data validation used:

- input BED: `/XCLabServer002_fastIO/gatk-faster-rnaseq/SRR949115_broad_hg38_run2/baseline/reference/exons.bed`
- reference dict: `/data1/pub/gatk/broad_hg38/Homo_sapiens_assembly38.dict`

Results:

- Direct `rust_interval_tools prepare` wall time: `3.91 s`.
- Python wrapper `step_bed_to_interval_list.py --backend rust` wall time: `0.64 s`.
- Python wrapper `step_split_hc_intervals.py --backend rust --scatter-count 8` wall time: `0.25 s`.
- Merged BED rows: `328,579`.
- Merged covered bases: `157,105,601`.
- Rust merged BED was byte-identical to the existing Python `step_build_merged_exon_bed.py` output.
- Merged interval_list rows: `328,579`.
- Merged interval_list covered bases: `157,105,601`.
- Dictionary/coordinate invalid intervals: `0`.
- 8 shards were produced.
- Shard total covered bases: `157,105,601`.
- Mean shard bases: `19,638,200.125`.
- Minimum shard bases: `19,637,370` (`0.004227%` below mean).
- Maximum shard bases: `19,639,886` (`0.008585%` above mean).
- Maximum single interval size: `347,300` bases (`1.7685%` of mean shard size).
- GATK `IntervalListToBed` accepted the Rust interval_list.
- A tiny GATK `HaplotypeCaller` smoke test using the Rust interval_list with `--interval-set-rule INTERSECTION` completed successfully.

Baseline comparison context:

- Prior GATK `BedToIntervalList` on the raw exon BED took about `6.258 s`.
- Prior manual validation found GATK `BedToIntervalList` on merged BED and GATK `SplitIntervals` on the full exon interval list expensive enough to interrupt.
- The Rust wrapper path for conversion plus 8-way splitting took about `0.89 s`, which is already more than `5x` faster than the recorded raw `BedToIntervalList` step alone.

## Conclusion

Module 1 is implemented and validated at the step level. The default pipeline remains GATK, and Rust interval preparation is available through opt-in wrapper and pipeline backend flags.
