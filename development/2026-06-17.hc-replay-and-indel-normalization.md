# HaplotypeCaller Rust replay tables and indel normalization

## Aim

Continue module 6 matching work against Java GATK `HaplotypeCaller` by adding Rust-side stage emission and making one algorithmic correction that follows GATK-style indel representation.

The comparison baseline remains the same-input GATK rerun:

`/XCLabServer002_fastIO/gatk-faster-rnaseq/SRR949115_broad_hg38_run2/module6_gatk_hc_rerun_20260616`

## Code changes

- Added `rust_hc_region_replay`.
  - Reuses the current Rust caller's pileup evidence path.
  - Emits active-region proxy, active-locus, read-observation, candidate-event, genotype, and placeholder haplotype/PairHMM TSV tables.
  - The PairHMM and haplotype tables are intentionally empty except headers because the current Rust caller still has no local assembly or PairHMM.
- Added `rust_hc_vcf_to_genotype_table`.
  - Converts GATK/Rust VCF records into the same genotype-stage TSV schema used by Rust replay.
  - This allows `rust_hc_genotype_diff` to compare final call fields using `chrom,pos,ref,alt`.
- Added reference-context left normalization for simple Rust pileup indels.
  - This fixes right-shifted homopolymer insertion/deletion representations such as Rust-private `T>TT` records that should match GATK's leftmost `A>AT` representation.
  - Unit tests cover homopolymer insertion and deletion normalization.
- Increased caller work partitioning to `8x` the requested thread count before Rayon scheduling.
  - This gives the 40-thread run more independent work units and slightly improves load balance on uneven RNA coverage.
- Updated `build.md` with the new binaries and matching commands.

## Conclusion

The replay/converter tools now make the current Rust caller auditable at candidate-event and final-genotype stages. Indel left normalization is a real algorithmic improvement and substantially reduces Rust-private indels, but the caller is still not close enough to Java GATK.

The remaining gap is not mostly indel representation anymore. Full PASS discordance is now dominated by:

- missing Java active-region assembly and haplotype generation;
- missing PairHMM read-vs-haplotype likelihoods;
- simplified allele likelihood marginalization and genotyping;
- many SNP private calls and genotype/QUAL/PL differences.

Next work should implement the Java-like active-region grouping/read finalization and then local haplotype assembly + PairHMM, rather than further pileup threshold tuning.
