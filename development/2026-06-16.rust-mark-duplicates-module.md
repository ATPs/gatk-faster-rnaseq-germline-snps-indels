# Rust MarkDuplicates module

## Aim

Develop roadmap module 5, `rust_mark_duplicates`, as a controlled alternative to Picard/GATK `MarkDuplicates` and `sambamba markdup` for the current coordinate-sorted RNA-seq BAM pipeline.

## Implemented scope

- Added `rust_mark_duplicates` as a Rust binary in `src/rust`.
- Wired `src/step_mark_duplicates.py --mode rust`.
- Added `run_pipeline.py --mode rust`, preserving the default `baseline` mode.
- Preserved input record order and wrote a coordinate-sorted BAM plus `.bai`.
- Implemented Picard-style duplicate keys for ordinary non-UMI paired-end and fragment records:
  - library from `@RG LB`;
  - stranded unclipped start;
  - reference id;
  - strand;
  - paired-end second-end key;
  - `SUM_OF_BASE_QUALITIES` scoring with minimum base quality 15.
- Skipped duplicate marking for unmapped, secondary, and supplementary records.
- Wrote Picard-compatible duplicate metrics columns.
- Implemented Picard-compatible `ESTIMATED_LIBRARY_SIZE` using the same bisection equation used by Picard duplication metrics.

Out of scope for v1:

- UMI-aware duplicate marking.
- Optical duplicate tagging and optical duplicate counts.
- Read removal modes.

## Build environment

The Rust package uses `rust-htslib`, whose `hts-sys` build uses bindgen. In the local conda base environment, the successful build/test command was:

```bash
source /data/p/anaconda3/bin/activate base
export PATH=/data/p/bin:$PATH
export LIBCLANG_PATH=/data/p/anaconda3/lib
export BINDGEN_EXTRA_CLANG_ARGS='-I/usr/lib/gcc/x86_64-redhat-linux/11/include -I/usr/include'
/data/p/sys/rust/1.96.0/bin/cargo fmt --manifest-path src/rust/Cargo.toml --all
/data/p/sys/rust/1.96.0/bin/cargo test --release --manifest-path src/rust/Cargo.toml --bin rust_mark_duplicates
/data/p/sys/rust/1.96.0/bin/cargo build --release --manifest-path src/rust/Cargo.toml --bin rust_mark_duplicates
```

Without the `LIBCLANG_PATH` and `BINDGEN_EXTRA_CLANG_ARGS` environment variables, bindgen could not find `libclang` or `stddef.h`.

## Conclusion

`rust_mark_duplicates` passes the module 5 acceptance criteria for the current `SRR949115` non-UMI coordinate-sorted RNA-seq BAM:

- synthetic paired-end duplicate flags and metrics match Picard;
- full-sample duplicate counts and key Picard metrics match Picard;
- final VCF concordance is better than the previous `sambamba` comparison;
- estimated end-to-end runtime improves versus the measured baseline.

The pipeline still defaults to `baseline`; use `--mode rust` to select this module explicitly.
