# Baseline and sambamba benchmark results

## Aim
Run the Broad hg38 RNA-seq germline SNP/indel pipeline on SRR949115 and compare the first optimization attempt: replacing Picard/GATK MarkDuplicates with sambamba markdup.

## Reference preparation
Built a STAR index from `/data1/pub/gatk/broad_hg38/Homo_sapiens_assembly38.fasta` and GENCODE v46 GTF with `--sjdbOverhang 99` at `/XCLabServer002_fastIO/gatk-faster-rnaseq/reference/broad_hg38_gencode_v46_readlen100_STAR.index`.

## Output comparison

- Baseline filtered VCF total variants: 41,151.
- Sambamba filtered VCF total variants: 41,156.
- PASS variants: baseline 36,435; sambamba 36,434.
- Non-PASS variants: baseline 4,716; sambamba 4,722.
- `bcftools isec` records: baseline-private 12, sambamba-private 17, shared 41,139.
- Deduplicated BAM `samtools flagstat` read and duplicate counts were identical:
  - total reads: 16,930,268;
  - duplicates: 1,204,446;
  - mapped reads: 16,930,268.

## Conclusion

For this SRR949115 test, sambamba markdup was only slightly faster than Picard MarkDuplicates, but downstream ApplyBQSR became much slower. The first optimization attempt is therefore not a net improvement. The measured bottlenecks to target next are HaplotypeCaller, SplitNCigarReads, and ApplyBQSR.
