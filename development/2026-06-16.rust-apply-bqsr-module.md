# Rust ApplyBQSR module

## Aim

Develop roadmap module 2, `rust_apply_bqsr`, as an opt-in replacement for GATK `ApplyBQSR` that consumes the GATK `BaseRecalibrator` table and rewrites BAM base qualities.

## Implementation

- Added `rust_apply_bqsr` under `src/rust/`.
- Added a shared Rust `apply_bqsr` implementation that:
  - parses GATK recalibration tables for the standard read-group, reported-quality, cycle, and context covariates;
  - applies substitution-quality recalibration with `--use-original-qualities`;
  - processes reads in ordered batches with Rayon worker threads, so `--threads` is used for read-level BQSR work while output order is preserved;
  - preserves coordinate order, header, read groups, flags, mate fields, and auxiliary tags except for GATK-style removal of `BI` and `BD`;
  - applies GATK `WellformedReadFilter` compatibility for CIGAR `N` records;
  - writes BAM plus `.bai`.
- Added `--backend gatk|rust`, `--rust-bin`, and `--threads` to `src/step_apply_bqsr.py`.
- Added `--apply-bqsr-backend` and `--rust-apply-bqsr` pipeline plumbing in `src/run_pipeline.py`.
- Kept the default backend as GATK.

The Python wrapper intentionally does not pass Rust `--use-report-quantization`, because the current GATK path uses the unquantized recalibrated qualities expected by the baseline output.

## Pre-Parallel Observation

The first compatible Rust implementation matched GATK `ApplyBQSR` at BAM quality and flagstat level for the full SRR949115 baseline input, but did not meet the roadmap adoption target of at least 2x faster than GATK. It was retained only as a correctness baseline before adding ordered batch-parallel read processing.

## Updated Conclusion

After ordered batch-parallel read processing, `rust_apply_bqsr` meets the roadmap speed target for this SRR949115 benchmark while preserving BAM-level parity with GATK `ApplyBQSR`.

Keep `rust_apply_bqsr` available as an opt-in validated backend. Do not switch the pipeline default from GATK until the downstream filtered VCF concordance check passes and at least one repeated full-run timing confirms the speedup.

Remaining acceptance work before any default replacement:

- downstream filtered VCF concordance against the GATK ApplyBQSR output;
- repeated full-run timings to rule out run-to-run noise;
- profiling of the Rust implementation if further optimization is desired.
