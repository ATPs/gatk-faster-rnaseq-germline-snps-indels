# Module 6 caller-side Rust prefilter validation

## Aim

Start roadmap module 6 without attempting a full `HaplotypeCaller` rewrite. The implemented experiment is a Rust caller-side interval prefilter that scans a recalibrated BAM for lightweight pileup evidence and emits GATK-compatible candidate intervals for the existing GATK `HaplotypeCaller`.

This follows the module 6 scope:

- allowed: interval/window prefiltering and lightweight pileup summaries;
- out of scope: replacing local assembly, graph pruning, genotyping, or annotations.

## Changes

- Added Rust binary `rust_hc_prefilter` under `src/rust/src/bin/`.
- Added Python wrapper `src/step_hc_prefilter.py`.
- Added `RUST_HC_PREFILTER` to `src/step_common.py`.
- Added optional pipeline arguments:
  - `--hc-prefilter-backend none|rust`;
  - `--rust-hc-prefilter`;
  - pileup thresholds for mapQ, baseQ, alt count, indel count, alt fraction, padding, max depth, supplementary reads, and empty-output behavior.
- The default pipeline behavior is unchanged: `--hc-prefilter-backend none`.

## Contract

`rust_hc_prefilter candidate-intervals`:

- reads a coordinate-sorted, indexed BAM;
- reads a FASTA through `.fai`;
- reads and preserves the GATK interval_list header;
- scans only the requested intervals;
- ignores unmapped, secondary, duplicate, and QC-fail reads;
- includes supplementary reads by default to match the observed GATK `HaplotypeCaller` read filters, with `--exclude-supplementary` available for stricter experiments;
- counts simple mismatch and indel-start evidence from pileups;
- writes padded, sorted, merged candidate intervals as `.interval_list`;
- clips padded candidate intervals back to the original input interval list before output, so `HaplotypeCaller -L` does not call padding-external exon-adjacent regions;
- optionally writes BED and a summary TSV.

The output is intended as an experimental input to GATK `HaplotypeCaller -L`, not as a VCF-producing caller.

## Conclusion

Module 6 now has a standalone Rust caller-side prefilter experiment wired into the Python pipeline behind an explicit opt-in backend. It produces GATK-compatible intervals and passed smoke tests plus a full SRR949115 clipped 5x8 scatter validation through `HaplotypeCaller`, `MergeVcfs`, `VariantFiltration`, and `ValidateVariants`.

The final clipped run clears the module 6 numeric acceptance thresholds:

- no malformed or unsorted VCF output after gather;
- PASS sensitivity `0.99975298`, above the `99.9%` target;
- HCPF-private PASS fraction `0.0165%`, below the `0.5%` general speed-first threshold;
- the missed baseline PASS calls were reviewed by GENCODE v46 region/consequence class;
- caller-side wall time through merge about `8:25` versus the baseline `HaplotypeCaller` wall time of `23:45.558`, meeting the `2x` target.

The optimization should remain opt-in for now because it is a speed-first caller-side filter with deliberate biological tradeoffs. The narrow module 6 acceptance criteria are met for SRR949115; adoption as the default should wait for repeated samples or cohort-level confirmation.
