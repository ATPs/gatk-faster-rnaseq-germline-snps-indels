# HaplotypeCaller base-quality threshold handling parity fix

## Aim

Bring Rust HaplotypeCaller PairHMM input quality handling closer to Java by replacing the current `max(18)` behavior with the Java thresholding rule.

## Source check

Local source verification used:

- Rust PairHMM read preparation in `src/rust/src/haplotype_caller.rs`
- Java quality preprocessing in `/data/p/gatk/gatk/src/main/java/org/broadinstitute/hellbender/tools/walkers/haplotypecaller/PairHMMLikelihoodCalculationEngine.java`
- Java quality constants in:
  - `/data/p/gatk/gatk/src/main/java/org/broadinstitute/hellbender/utils/pairhmm/PairHMM.java`
  - `/data/p/gatk/gatk/src/main/java/org/broadinstitute/hellbender/utils/QualityUtils.java`

Confirmed findings:

- Rust was using `quals[i].min(mapq).max(18)` in both production and replay PairHMM read preparation.
- Java first caps base qualities by mapping quality, then applies `setToFixedValueIfTooLow(...)`.
- Java `PairHMM.BASE_QUALITY_SCORE_THRESHOLD` is `18`.
- Java `QualityUtils.MIN_USABLE_Q_SCORE` is `6`.
- Therefore Java does not raise low base qualities to 18. It converts any post-MAPQ-capped value below 18 to 6.
- Java also floors insertion and deletion open penalties to the same minimum usable quality.

Conclusion from source: the Rust caller was overstating low-quality read evidence in PairHMM input.

## Code changes

Changed file:

- `src/rust/src/haplotype_caller.rs`

Implementation:

- Added named constants for:
  - PairHMM base-quality threshold `18`
  - minimum usable quality `6`
- Added `pair_hmm_base_quality(base_quality, mapq)` that:
  - caps by MAPQ;
  - converts values below 18 to 6.
- Added `pair_hmm_indel_open_quality(quality)` that floors BI/BD-derived penalties to 6.
- Replaced the four production/replay `quals[i].min(mapq).max(18)` call sites with the new helper.
- Applied the indel-quality helper to BI/BD tag values in the same production/replay PairHMM read-preparation paths.
- Added focused unit tests that verify:
  - MAPQ capping still happens;
  - values below the threshold are squashed to 6, not raised to 18;
  - BI/BD-derived low values are floored to 6.
