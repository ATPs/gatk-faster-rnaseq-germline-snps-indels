# Good-set filter correction for single-sample diploid VCFs

## Aim

Fix the round1 comparison workflow so the "golden good-variant set" is non-empty and matches what the user intended by a minimum-support filter.

## Source check

Local output verification used:

- `module6_good_compare_20260617_round1/java.split.vcf.gz`
- `module6_good_compare_20260617_round1/rust.split.vcf.gz`
- `module6_good_compare_20260617_round1/java_vs_rust.good.summary.tsv`

Confirmed findings:

- The first round used:
  - `FILTER == PASS`
  - `INFO/DP >= 10`
  - `INFO/AC >= 3`
- That produced empty `java.good.vcf.gz` and `rust.good.vcf.gz`, and therefore a zero-row comparison.
- Querying the split Java VCF showed that `INFO/AC` only takes values `1` or `2`:

```text
1
2
```

- This is expected for a single-sample diploid VCF after splitting multi-ALT records.
- The actual per-allele support signal the user wanted is represented by sample-level ALT depth, i.e. `FORMAT/AD[0:1]`.

## Changes

Updated:

- `development/2026-06-17.concrete-implementation-plan.md`
- `development/2026-06-17.module6-good-compare-round1.zsh`

Correction:

- Replaced the impossible `INFO/AC>=3` rule with:
  - `FORMAT/DP[0] >= 10`
  - `FORMAT/AD[0:1] >= 3`

This keeps the good set allele-level after splitting and enforces the intended minimum ALT read support.

## Verification

Checks run:

```bash
bcftools query -f '%INFO/AC\n' java.split.vcf.gz | sort -nu
bcftools view -H -f PASS -i 'FORMAT/DP[0]>=10 && FORMAT/AD[0:1]>=3' java.split.vcf.gz | head
bcftools view -H -f PASS -i 'FORMAT/DP[0]>=10 && FORMAT/AD[0:1]>=3' rust.split.vcf.gz | head
bcftools view -H -f PASS -i 'FORMAT/DP[0]>=10 && FORMAT/AD[0:1]>=3' java.split.vcf.gz | wc -l
bcftools view -H -f PASS -i 'FORMAT/DP[0]>=10 && FORMAT/AD[0:1]>=3' rust.split.vcf.gz | wc -l
```

Observed counts on the existing round1 split files:

- Java-good candidates: `12,327`
- Rust-good candidates: `14,229`

## Conclusion

The original `AC>=3` rule was not usable for this dataset.

The corrected good-set rule is now:

- `PASS`
- `FORMAT/DP[0] >= 10`
- `FORMAT/AD[0:1] >= 3`

The next step is to rerun only the good-set filtering / compare / sampling tail of round1 using the already generated split VCFs.
