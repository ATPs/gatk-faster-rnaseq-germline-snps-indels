# Build Guide Generalization

## Aim

Move the server-specific build instructions out of the repository root and replace them with a generic `build.md` that is useful for contributors on other machines.

## Changes

- Moved the original root `build.md` content to `development/2026-06-18.server-specific-build.md`.
- Added a note there explaining that it is an archived server-specific document.
- Replaced the root `build.md` with a portable guide that:
  - explains the repository layout
  - documents general prerequisites
  - points users to review `src/rust/.cargo/config.toml` for local path overrides
  - uses a normal `cargo build --release --bins` flow as the default
  - explains how to install binaries into `rust_binary`
  - shows generic verification commands for both Rust and Python entrypoints

## Conclusion

The root build guide now reads as contributor documentation instead of local server notes, while the original environment-specific commands remain preserved in `development/` for reference.
