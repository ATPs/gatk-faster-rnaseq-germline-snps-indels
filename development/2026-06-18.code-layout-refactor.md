# Professional Code Layout Refactor

## Aim

Reorganize the Python and Rust code into a more professional layout now that feature development is mostly complete, keep implementation files below roughly 1000 lines where practical, add a real top-level Python entrypoint, support skipping pipeline steps with explicit artifact reuse, and preserve current behavior through verification.

## Changes

### Python layout

- Added the repository entrypoint: `gatk-faster-rnaseq-germline-snps-indels.py`.
- Moved real Python implementation into the package `src/gatk_faster_rnaseq/`.
- Split pipeline control and step implementations into:
  - `src/gatk_faster_rnaseq/cli.py`
  - `src/gatk_faster_rnaseq/runtime.py`
  - `src/gatk_faster_rnaseq/pipeline/runner.py`
  - `src/gatk_faster_rnaseq/steps/*.py`
- Removed the old top-level wrapper files under `src/` so the package layout is the only maintained Python implementation path.

### Pipeline controls

- Added per-step skip flags:
  - `--skip-star-align`
  - `--skip-mark-duplicates`
  - `--skip-split-n-cigar`
  - `--skip-base-recalibrator`
  - `--skip-apply-bqsr`
  - `--skip-hc-prefilter`
  - `--skip-haplotype-caller`
  - `--skip-variant-filtration`
- Kept `--skip-bqsr` as a shortcut for skipping both BQSR steps.
- Added explicit artifact reuse inputs:
  - `--aligned-bam`
  - `--dedup-bam`
  - `--split-bam`
  - `--recal-table`
  - `--recal-bam`
  - `--raw-vcf`
- Added validation so a skipped step must have the required downstream input supplied.

### Rust layout

- Reorganized Rust implementation so helper logic lives in library modules under `src/rust/src/`.
- Moved Rust executable entrypoints to `src/rust/cli/*.rs` so they are separated from library modules without using the misleading `src/rust/src/bin` source layout.
- Split oversized Rust files into focused modules, including:
  - `base_recalibrator/`
  - `apply_bqsr/`
  - `split_n_cigar/`
  - `hc_tools/`
  - `haplotype_caller/`
- Replaced the previous monolithic module files with submodules and a smaller `src/rust/src/lib.rs`.
- Fixed the post-split `haplotype_caller` test-module wiring so all tests still run after the file split.

### Project guidance

- Updated `AGENTS.md` with the new organization rules:
  - main Python entrypoint is the repository-root script
  - real Python code belongs in `src/gatk_faster_rnaseq/`
  - do not reintroduce top-level pipeline wrappers under `src/`
  - Rust implementation belongs in library modules under `src/rust/src/`
  - Rust CLI entrypoints belong in `src/rust/cli/` and should remain thin
  - keep new Python and Rust source files below roughly 1000 lines when practical
- Updated `build.md` to reflect the new entrypoint and verification commands.

## Conclusion

The Python pipeline can skip individual steps safely when the required intermediate artifact is provided, and the Rust executable layout now follows the expected pattern of thin binaries over library modules. Verification passed after the refactor, including the real-data smoke run and the full Rust release test suite.
