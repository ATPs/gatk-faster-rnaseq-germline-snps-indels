# Rust binary auto fallback and downstream benchmark

## Aim

Record the current Rust binary layout, pipeline fallback behavior, and the first full downstream-only benchmark that reuses the existing STAR-aligned BAM instead of rerunning alignment.

This note reflects the current project direction:

- keep all Rust code in the single crate under `src/rust/`;
- keep shared Rust modules under `src/rust/src/`;
- keep one binary entrypoint per module under `src/rust/src/bin/`;
- install release binaries to `src/rust/bin/`;
- let Python prefer Rust automatically when the installed binary exists, and fall back to GATK otherwise.

Updated orchestrator:

- `src/run_pipeline.py`
- `src/step_common.py`

Current behavior:

- default Rust binary directory is `src/rust/bin/`;
- `run_pipeline.py` accepts `--rust-bin-dir` to override the binary directory;
- `run_pipeline.py` accepts `--no-rust` to force all Rust backends off;
- each Rust-capable wrapper accepts an explicit `--rust-bin`;
- backend mode `auto` prefers Rust only if the expected binary exists;
- otherwise the wrapper or pipeline falls back to the GATK path;
- `--aligned-bam` skips `star_align` and reuses an existing coordinate-sorted BAM.

This is the intended benchmark mode for downstream-only comparisons. Re-running STAR adds noise and does not help compare downstream modules when the aligned BAM is already fixed.

## Interpretation

The current parameter-only tuning path is no longer the main source of gain. The measured speedup now comes from replacing several heavy preprocessing stages with Rust and using interval scatter for the unchanged GATK caller.

What looks ready:

- `rust_interval_tools`
- `rust_split_n_cigar_reads` fast mode
- `rust_base_recalibrator`
- `rust_apply_bqsr`

What still needs caution:

- `rust_mark_duplicates` is fast, but the recalibrated BAM duplicate count still differs from baseline by 2 records;
- `rust_hc_prefilter` improved caller-stage wall time overall, but by itself it still costs `240.516 s`, so module 5 or caller-side work may deserve more attention than additional parameter tweaking.

## Current conclusion

The project now has the requested binary layout and pipeline behavior:

- Rust code stays under `src/rust/src`;
- release binaries are installed to `src/rust/bin`;
- the pipeline auto-detects Rust binaries and falls back to GATK when they are absent;
- downstream benchmarking can skip STAR cleanly with `--aligned-bam`.

On the current SRR949115 downstream-only benchmark with 40 threads, the Rust-backed path reduced wall time from `3194.727 s` to `655.029 s`, about `4.88x` faster than the baseline downstream subtotal while keeping PASS concordance at `99.8381%`.

The next development focus should be:

1. tighten `rust_mark_duplicates` parity;
2. profile and improve `rust_hc_prefilter`;
3. repeat the full downstream benchmark at least once more to rule out run-to-run noise before changing any pipeline defaults.
