# Standard pipeline and sambamba timing plan

## Aim

Run a reproducible baseline RNA-seq germline SNP/indel pipeline on `SRR949115`, record per-step wall clock time, then compare the first optimized version that replaces Picard/GATK `MarkDuplicates` with `sambamba markdup`.

## Current implementation

Added `src/run_pipeline.py`, a Python argparse runner that:

- uses the local tool paths documented in `AGENTS.md`;
- defaults to Broad hg38 resources under `/data1/pub/gatk/broad_hg38`;
- accepts FASTQ input or an existing STAR coordinate-sorted BAM;
- writes command logs under `logs/`;
- writes `timings.tsv` with `step`, `status`, seconds, minutes, command, and message;
- supports two modes:
  - `baseline`: GATK/Picard `MarkDuplicates`;
  - `sambamba`: `sambamba markdup` and `sambamba index`;
- generates exon intervals from the GENCODE GTF and reference `.fai`;
- runs STAR two-pass alignment, duplicate marking, `SplitNCigarReads`, optional BQSR, `HaplotypeCaller`, and `VariantFiltration`.

## WARP RNAWithUMIsPipeline note

`warp/pipelines/wdl/rna_seq/RNAWithUMIsPipeline.wdl` is an RNA/UMI processing and QC workflow. It runs FASTQ/uBAM conversion, UMI extraction, adapter clipping, STAR, UMI-aware duplicate marking, RNA metrics, alignment metrics, and contamination estimation. It does not contain `SplitNCigarReads`, `BaseRecalibrator`, `ApplyBQSR`, `HaplotypeCaller`, or `VariantFiltration`, so variant calling still needs to be appended after its aligned BAM output.

## Important assumptions

- The current default FASTA is `/data1/pub/gatk/broad_hg38/Homo_sapiens_assembly38.fasta`.
- The default known-sites are Broad hg38 `Homo_sapiens_assembly38.dbsnp138.vcf.gz`, `Homo_sapiens_assembly38.known_indels.vcf.gz`, and `Mills_and_1000G_gold_standard.indels.hg38.vcf.gz`.
- The STAR index should be built from the same Broad FASTA used by GATK. The previous GENCODE GRCh38.p14 STAR index was rejected because its contig set differs from Broad assembly38 alt/decoy contigs.
- The default STAR index path is now `/XCLabServer002_fastIO/gatk-faster-rnaseq/reference/broad_hg38_gencode_v46_readlen100_STAR.index`, built with `--sjdbOverhang 99` for the SRR949115 read length.
- The supplied GENCODE v46 GTF is used to generate exon intervals and contains `chr*` contig names compatible with Broad hg38 primary contigs.

Timing smoke run without known-sites:

/data/p/anaconda3/bin/python gatk-faster-rnaseq-germline-snps-indels/src/run_pipeline.py \

## Optimization direction

After collecting `timings.tsv` for baseline and sambamba modes:

1. Compare per-step time and identify the largest remaining bottleneck.
2. Prefer existing faster tools for commodity BAM/VCF operations before writing Rust.
3. Write Rust replacements only for a measured bottleneck with a narrow, testable contract.
4. Store Rust code under `src/` as required by `AGENTS.md`.
