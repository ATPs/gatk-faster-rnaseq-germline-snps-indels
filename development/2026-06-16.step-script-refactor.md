# Split pipeline steps into standalone scripts

## Aim

Refactor `src/run_pipeline.py` so it remains a single pipeline entrypoint, while each major step is available as an independently runnable script for isolated benchmarking and parallel testing.

## Changes

Added shared helper module:

- `src/step_common.py`

Added standalone step scripts:

- `src/step_build_exon_bed.py`
- `src/step_build_merged_exon_bed.py`
- `src/step_bed_to_interval_list.py`
- `src/step_star_align.py`
- `src/step_mark_duplicates.py`
- `src/step_split_n_cigar.py`
- `src/step_base_recalibrator.py`
- `src/step_apply_bqsr.py`
- `src/step_split_hc_intervals.py`
- `src/step_haplotype_caller.py`
- `src/step_merge_vcfs.py`
- `src/step_variant_filtration.py`

Updated `src/run_pipeline.py`:

- removed the temporary multi-component CLI mode so `run_pipeline.py` is again pipeline-only;
- kept merged-exon and scattered `HaplotypeCaller` support;
- changed pipeline steps to call the new standalone scripts instead of embedding all stage logic in one file;
- kept timing and log behavior in the pipeline orchestrator.

## Verification

Checked with:

- `python -m py_compile gatk-faster-rnaseq-germline-snps-indels/src/*.py`
- `python gatk-faster-rnaseq-germline-snps-indels/src/run_pipeline.py --help`
- `python gatk-faster-rnaseq-germline-snps-indels/src/step_haplotype_caller.py --help`

## Conclusion

The code structure now matches the intended workflow better:

- `run_pipeline.py` is the full-pipeline orchestrator;
- each heavy step can be run directly for isolated testing;
- future benchmarking can target one step at a time without adding a second control mode inside the pipeline runner itself.
